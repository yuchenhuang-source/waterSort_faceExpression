# ArrowPuzzle 玩法概要设计与数据结构说明（Design Doc）

版本：Draft v1  
日期：2026-02-04  
范围：玩法规则（Rule Layer）+ 关卡数据（Data Layer）+ 视觉样式规范（Style Spec）  
不包含：具体技术选型、渲染/引擎实现细节（由项目框架决定）

---

## 1. 游戏概述

**类型**：点击解谜（Click-to-Exit Puzzle）  
**核心体验**：在有限棋盘内，箭头（粗管道状路径）互相阻挡。玩家需要按正确顺序点击，让每支箭头沿其朝向“飞出棋盘”，最终清空所有箭头。

**核心卖点**：
- 规则简单（点击 → 退出/失败反馈）
- 解法依赖顺序与空间阻挡关系（轻量但有深度）
- 关卡易于数据化与编辑器化（路径即结构）

---

## 2. 关键术语

- **棋盘（Board）**：`width × height` 的二维网格。
- **格子索引（Index）**：将网格扁平化的一维索引 `0..(width*height-1)`。
- **箭头（Arrow）**：由一串格子索引 `indices[]` 构成的**正交折线路径**，并在末端绘制箭头头部。
- **占用（Occupancy）**：某格子是否被某箭头路径占用。
- **阻挡（Blocking）**：箭头尝试退出时，其前方通道被其他箭头占用导致无法退出。

---

## 3. 棋盘与坐标体系

### 3.1 index 与 (row, col)
- `row = floor(index / width)`
- `col = index % width`
- `index = row * width + col`

### 3.2 相邻关系（四邻接）
关卡中所有路径连接均为四邻接（上下左右）。对任意相邻两点 `a,b`：
- `|row(a)-row(b)| + |col(a)-col(b)| == 1`

> 说明：必须额外避免“跨行 +1/-1”造成的假相邻（例如从 col=width-1 到下一行 col=0）。

---

## 4. 箭头的数据语义（与给定关卡格式一致）

### 4.1 Arrow = 折线路径（Polyline on Grid）
每支箭头由一串 `indices` 表示：
- `indices.length >= 2`
- 相邻 indices 必须四邻接
- 同一支箭头内部建议不重复格子（通常不需要自交）

该序列按顺序连接格子中心，形成折线；再将折线“加粗”绘制成管道状实体。

### 4.2 箭头朝向（飞出方向）
箭头的“朝向/飞出方向”由路径最后一段决定：
- `a = indices[n-2]`，`b = indices[n-1]`
- `b` 相对 `a` 的方向即 `dir`

这使得关卡数据不必额外存 `dir`。

## 5. 玩法规则（Rule Layer）

> 本文以最常见的“点击→整支箭头尝试退出棋盘”为主。若你需要“滑动推进”变体，可在 5.6 扩展。

### 5.1 玩家操作
- 玩家点击（或触控）一支箭头。
- 若该箭头满足退出条件，则执行退出（动画）并从关卡移除。
- 若不满足，给出失败反馈（抖动/闪烁/音效）。

### 5.2 占用与互斥
- 每个格子在同一时刻最多只能被一支箭头占用。
- 不同箭头的 `indices` 在初始状态必须互不重叠。

### 5.3 阻挡（Blocking）判定思想
判定“能否退出”的关键在于：箭头末端朝向 `dir` 的前方通道是否被占用。

### 5.4 退出条件（Exit Condition）——推荐默认模式 A
为避免歧义，建议把退出规则做成可配置项：

**Mode A：直通出界（推荐默认）**
- 从箭头末端开始，沿 `dir` 方向一直检查到出界。
- 若通道上任一格被其他箭头占用，则该箭头不可退出。
- 若直到出界均为空，则可退出。

**Mode B：只看一格（更宽松）**
- 仅检查末端方向相邻 1 格是否为空。

> 你提供的关卡数据结构对 A/B 都适配；A 更符合“互相阻挡”的解谜感。

### 5.5 动画与状态
- 退出时播放“整支箭头沿 dir 飞出屏幕”的动画。
- 动画期间建议锁定输入（或做队列），避免状态竞争。
- 动画结束后：
  - 从 `arrows[]` 移除该箭头
  - 更新占用表
  - 触发胜利判断

### 5.6 胜利/失败
- **胜利**：`arrows.length == 0`
- **失败**：无硬失败；若无任何箭头可退出则为“死局”，UI 可提示重玩/撤销。

### 5.7 可扩展规则（可选，不作为 v1 必需）
- **滑动推进 Mode C**：点击后箭头先滑至阻挡前，再继续解锁。需要定义“路径整体如何移动与占用更新”，复杂度明显上升。
- **撤销/重做**：记录每步移除序列与状态快照。
- **提示**：基于当前状态搜索下一步可退出箭头。

## 6. 关卡数据结构（Data Layer）

### 6.1 顶层结构

```json
{
  "meta": {
    "version": "1.0",
    "created": "2026-02-04T07:26:30.403Z",
    "levelName": "未命名关卡",
    "author": "",
    "description": ""
  },
  "config": {
    "width": 5,
    "height": 5
  },
  "arrows": [
    { "indices": [10, 11, 16, 17, 18, 23] }
  ]
}
```

### 6.2 字段定义

#### meta
- `version`：关卡格式版本号
- `created`：创建时间（ISO 8601）
- `levelName/author/description`：编辑器与展示用

#### config
- `width, height`：棋盘尺寸
- （可选扩展）`rule.exitMode`：A/B/C 等

#### arrows[]
- `indices: number[]`：箭头折线路径格子索引序列
- （可选扩展）`id/style`：编辑器引用与皮肤

### 6.3 运行时派生数据（不建议写入关卡文件）
- `dir`：由 indices 尾段推导
- `occupied[]`：格子占用表（index → arrowId）
- `points[]`：用于渲染的点缓存（格子中心点序列）

## 7. 箭头视觉样式规范（Style Spec）

基于你截图中表现，箭头为“粗圆角管道 + 三角箭头头”的扁平化图标风格。

### 7.1 风格关键词
- 实心纯色（黑/近黑）
- 粗线条（带宽路径）
- 圆角拐弯（round join）
- 末端三角箭头头（与主体连成一体）

### 7.2 推荐参数（相对 cell 尺寸）
假设每格边长为 `cell`：
- 主体厚度 `thickness = 0.25 ~ 0.35 * cell`
- 拐角：圆角连接（round join）
- 端帽：圆端（round cap）
- 箭头头部：
  - 长度（沿 dir）`headLen = 0.45 ~ 0.6 * cell`
  - 底边宽 `headWidth ≈ 1.2 ~ 1.8 * thickness`
- 颜色：默认 `#111` / `#000`
- 背景：浅灰（便于对比）

### 7.3 命中区域（交互友好）
- 触控端建议命中区域略大于视觉（例如外扩 6~12px 或 0.1*cell），以提升可点中性。

---

## 8. 关卡合法性与编辑器校验建议

### 8.1 结构校验（导入/保存必做）
- 字段必填、类型正确
- `width,height >= 1`
- `indices.length >= 2`

### 8.2 语义校验（强烈建议）
- 所有 `index` 满足：`0 <= index < width*height`
- 每条 arrow 相邻点必须四邻接
- 不同 arrow 之间不得占用同一格
- （可选）至少存在一个可退出箭头（按 exitMode）
- （可选）可解性验证

---

## 9. 示例解析（用于理解数据含义）

以示例 `indices: [10, 11, 16, 17, 18, 23]`，在 `width=5` 下：
- 10→11：向右
- 11→16：向下（+5）
- 16→17→18：向右
- 18→23：向下（+5）

因此这是一个“右→下→右→下”的折线管道箭头，末段向下，朝向为 **Down**。
