# Image Import Notes

目标：把"透明背景 + 彩色线条"的简笔画图片自动转成编辑器关卡数据（矩形网格 + 彩色箭头）。

## 当前需求与约束
1. **透明像素 = 无路径**。
2. **彩色线条 = 路径**，颜色用于区分箭头类型（需要保留颜色）。
3. **线条宽度一致**，不要求 1px。
4. **网格尺寸需要自动推断**（基于线宽与线条间距）。
5. **一条线 = 一个箭头**：保留所有弯折，不做单调拆分。
6. **箭头之间不能相邻**：任何两条箭头路径都不能有 4-连通的相邻格子。
7. **箭头头部**：仅在路径两端检测（比较端点与线条主体的局部密度）。

## 当前实现位置
脚本：`scripts/image_import.py`（v6）

输出示例：
- `test/import_result_v6.json`
- `test/import_result_v6_step*.png`（调试可视化）

备份：`scripts/image_import_v5.1_backup.py` (v5.1) 和 `scripts/image_import_v6.py` (原始v6)

## 算法流程（v6 – "路径优先，保留弯折"）

### 核心原则：连通性第一，颜色第二，一线一箭

**v4/v5 问题**：先按颜色分组 → 同一线因色差断裂；单调拆分 → 一线拆成多箭头。

**v6 解决方案**：
1. 骨架拓扑决定连通性（忽略颜色）
2. 先追踪完整线条，再统一上色
3. 相邻路径合并验证
4. **不做单调拆分** → 一条物理线 = 一个箭头（保留所有弯折）

### Phase 1: 预处理
1. **读取 RGBA**，`alpha >= threshold` 得到笔画 mask。
2. **黑色边框过滤**：亮度 < 30 的像素视为描边（非线条主体）。
3. **估计线宽**：距离变换统计，取峰值直径。
4. **形态学闭运算**：半径 = `line_width * 0.7`，填充线条内微小缝隙。

### Phase 2: 颜色预处理
5. **中心采样 + KMeans**：DT 高值像素 → 肘部法自动选 k。
6. **生成 label_img**：仅原始像素（边缘色差不传播到闭运算像素）。

### Phase 3: 网格估计
7. **合并骨架** → Zhang-Suen 细化（忽略颜色，连续性最优）。
8. **自相关周期检测** → 网格步长 X/Y。
9. **相位加权直方图 + 穷搜** → 网格偏移优化。

### Phase 4: 像素级路径追踪
10. **骨架 8-连通邻居图** → 端点开始 DFS → 有序像素序列。
11. **过滤短碎片** < `line_width` 像素。
12. 结果：**~240 条独立像素路径**（每条 = 一段连续骨架）。

### Phase 5: 端点检测 + 网格吸附
13. **端点箭头头检测**：
    - 仅检测两端
    - 局部窗口密度 vs 线条主体密度（阈值1.4×）
    - 确定路径方向（箭头头在末端）
14. **网格吸附**：
    - 可选 Douglas-Peucker 简化
    - 像素 → 格子映射
    - 4-连通插值补全
15. **整条路径上色**：中心像素投票 → 唯一颜色。
16. 结果：**~140 条网格路径**（带颜色）。

### Phase 6: 相邻路径合并（关键验证步骤）
17. **检测 4-相邻路径**：如果两条路径有相邻格子 → 应该是一条线被误拆。
18. **端点连接合并**：同色 + 端点相邻 → 合并为一条。
19. **迭代直到稳定**。
20. 结果：**~98 条最终线条**（合并了 ~40 条）。

### Phase 7: 箭头输出
21. **保留完整弯折**：不做单调拆分，直接输出完整线条。
22. **长度过滤**：`< min_arrow_len` 的丢弃。
23. 输出 JSON + debug PNG。

### 一线一箭验证
- 98 条物理线 → **98 个箭头** ✓
- 箭头之间无相邻格子 ✓

## v4/v5 → v6 核心架构变化

| 方面 | v4/v5 | v6 |
|------|------|-----|
| **连通性来源** | 格子密度检测/颜色分组 | **骨架拓扑（忽略颜色）** |
| **路径提取** | 网格格子 → 边追踪 | **像素骨架 → DFS 追踪** |
| **颜色分配** | 先分色 → 提路径 | **先提路径 → 后上色** |
| **单调拆分** | 1条线 → N个箭头 | **1条线 = 1个箭头** |
| **相邻检测** | 无 | **合并相邻路径（验证）** |
| **箭头头检测** | 全路径密度检测 | **仅两端局部对比** |
| **最终箭头数** | 70-289 | **98 (= 98条线)** |
| **平均箭头长** | 4.7-7.7 | **19.4 格** |
| **≤3格短箭头** | 67-110 | **6 个** |

### 解决的核心问题
1. ❌ 不相连线条被误连 → ✅ 骨架 CC 天然分离
2. ❌ 相连线条因色差断裂 → ✅ 先追踪后上色
3. ❌ 转角被误判为箭头头 → ✅ 仅检测两端
4. ❌ 一线拆成多箭头 → ✅ 保留完整弯折

## 可调参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--colors` | 0 (auto) | 颜色聚类数（0=自动检测） |
| `--alpha-threshold` | 1 | 透明度阈值 |
| `--close-radius` | -1 (auto) | 形态学闭运算半径（0=关闭，-1=自动） |
| `--force-step-x` | 0 (auto) | 强制网格步长 X |
| `--force-step-y` | 0 (auto) | 强制网格步长 Y |
| `--min-arrow-len` | 2 | 最短箭头长度（格子数） |
| `--border-brightness` | 30 | 黑色边框亮度过滤阈值（0=关闭） |
| `--simplify-epsilon` | 0.0 | Douglas-Peucker 简化阈值（0=关闭） |
| `--debug` | false | 生成调试可视化图片 |

## 如何运行

```bash
# 基本运行
python scripts/image_import.py --input test/美人鱼.png --out test/import_result.json

# 调试可视化
python scripts/image_import.py --input test/美人鱼.png --out test/import_result_v6.json --debug

# 手动指定颜色数
python scripts/image_import.py --input test/美人鱼.png --colors 3 --debug

# 启用 Douglas-Peucker 简化（减少路径点数）
python scripts/image_import.py --input test/美人鱼.png --simplify-epsilon 1.5 --debug

# 强制网格步长
python scripts/image_import.py --input test/美人鱼.png --force-step-x 40 --force-step-y 40 --debug
```

## 已知限制 & 后续建议
1. **不支持环形箭头**：DFS 从端点追踪，环形会被任意切断。
2. **网格步长依赖骨架**：极细/极粗线条可能需要 `--force-step`。
3. **箭头头检测基于密度**：无明显三角的箭头可能检测失败（影响方向）。
4. **建议**：导入后在编辑器中验证箭头方向。
