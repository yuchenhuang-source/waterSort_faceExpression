```bash
cd /Users/yuchenhuang/Downloads/playables/ballsort/ballsort-multi--液体球效果
```

### 2. 浏览器测试（桌面端）

#### 2.1 打开游戏
在浏览器中访问: `http://localhost:5173`

#### 2.2 打开性能监控
1. 按 `F12` 打开Chrome DevTools
2. 切换到 `Performance` 标签
3. 点击录制按钮（圆圈图标）
4. 进行以下操作：
   - 点击试管移动球
   - 连续点击多次
   - 等待动画完成
5. 停止录制
6. 查看FPS图表，确认是否稳定在60fps

#### 2.3 实时FPS监控
在浏览器控制台（Console）中运行：

```javascript
// 显示实时FPS
let lastTime = performance.now();
let frames = 0;
function showFPS() {
    frames++;
    const now = performance.now();
    if (now >= lastTime + 1000) {
        const fps = Math.round((frames * 1000) / (now - lastTime));
        console.log(`FPS: ${fps}`);
        frames = 0;
        lastTime = now;
    }
    requestAnimationFrame(showFPS);
}
showFPS();
```

## 性能对比测试

### 测试场景

| 场景 | 测试内容 | 预期FPS | 优化前FPS |
|------|---------|---------|----------|
| 开场加载 | 游戏启动到可交互 | 60fps | 24fps |
| 单球移动 | 点击移动1个球 | 60fps | 30-40fps |
| 多球移动 | 连续移动3个球 | 55-60fps | 20-30fps |
| 水位上升 | 球落地水位上升动画 | 60fps | 25-35fps |
| 完成动画 | 试管完成特效 | 55-60fps | 30-40fps |

### 关键指标

#### 1. 主线程任务时长
- **优化目标**: 单帧任务 < 16ms（60fps）
- **查看方式**: Performance录制 → Main线程 → 查看任务时长

#### 2. drawLiquid调用次数
- **优化前**: 初始化约96次，动画期间约30次/秒
- **优化后**: 初始化14次，动画期间约20次/秒

#### 3. 内存使用
- **查看方式**: Performance Monitor → JS Heap Size
- **预期**: 稳定在50-100MB，无持续增长

## 性能问题排查

### 如果FPS仍然低于50fps

#### 1. 检查设备性能
```javascript
// 在控制台运行
console.log('User Agent:', navigator.userAgent);
console.log('Hardware Concurrency:', navigator.hardwareConcurrency);
console.log('Device Memory:', navigator.deviceMemory, 'GB');
```

#### 2. 检查WebGL是否启用
```javascript
// 在控制台运行
const canvas = document.querySelector('canvas');
const gl = canvas.getContext('webgl') || canvas.getContext('experimental-webgl');
console.log('WebGL Enabled:', !!gl);
if (gl) {
    console.log('WebGL Renderer:', gl.getParameter(gl.RENDERER));
}
```

#### 3. 检查draw call次数
在Chrome DevTools → Performance录制中：
- 查看 `GPU` 部分
- 确认draw call次数是否合理（应 < 100次/帧）

#### 4. 检查长任务
在Performance录制中：
- 查找红色三角标记（长任务警告）
- 点击查看具体耗时函数
